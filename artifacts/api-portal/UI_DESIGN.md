# AI Gateway — 前端 UI 核心设计规范

> 供后续 agent 参考，任何涉及前端样式的修改都应以本文档为准。

---

## 一、整体基调

- **暗色系**：全局深色背景，低饱和度 + 少量亮色点缀
- **玻璃质感**：卡片与行项使用半透明背景叠加，不使用纯色实心块
- **克制的颜色**：除 provider 标识色外，尽量避免大面积高饱和度颜色
- **双密度模式**：默认使用舒适密度；可切换紧凑密度以提升信息承载量

---

## 二、字号规范（标题三级制：22px / 18px / 14px，次级 12px）

| 场景 | 字号 | 字重 |
|---|---|---|
| 页面主标题（Header H1，如 "AI Gateway"） | `22px` | `700` |
| 卡片节标题（SectionTitle / H2） | `18px` | `700` |
| 内容节标题（H3，如各功能小节）| `14px` | `700` |
| 子节标题（H4，如 SDK 示例分语言标题） | `12px` | `600` |
| 分组/厂商按钮标题（如 OpenAI、OpenRouter） | `14px` | `600` |
| 子分组标题（如 OpenRouter 子厂商） | `14px` | `600` |
| **主要内容行**（模型 ID、Header 字段名/值、SDK 版本） | **`14px`** | `400`（代码体）/ `600`（版本号） |
| 描述文字、说明 | `14px` | `400` |
| 辅助元数据（模型数量、Context 窗口大小等） | `12px` | `400` |
| 微型标签/徽章（Badge、context tag） | `12px` | `600–700` |
| 提示文字（placeholder、italic hint） | `12px` | `400`，斜体 |

**标题层级原则**：
- H1（页面主标题）= `22px`，仅用于顶部 Header 的产品名，此字号固定不变
- H2（区块标题 SectionTitle）= `18px`，用于页面内各大功能卡片的分区标题
- H3（内容节标题）= `14px`，用于卡片内具体小节的标题（与正文同尺寸，以字重 700 和下边框区分）
- H4（子节标题）= `12px`，用于 H3 下的进一步细分

**正文原则**：普通内容默认使用 `14px`；次级说明、元数据、提示、标签使用 `12px`。不使用 `12px` 以下字号，除非是极少数不可避免的密集图表刻度。

## 三、字体族

| 类型 | 字体 |
|---|---|
| 正文 / 界面文字 | 系统字体栈（默认） |
| 代码 / 模型 ID / HTTP Headers / 版本号 | `Menlo, monospace` |

所有模型 ID、HTTP header 字段名和字段值、SDK 版本号，统一使用 `Menlo, monospace`。

## 四、颜色规范

### 4.1 文本颜色层次

| 层次 | 色值 | 用途 |
|---|---|---|
| 强调 / 标题 | `#e2e8f0` | 卡片主标题、重要标签 |
| 主要内容 | `#cbd5e1` | 分组标题文字、模型组名 |
| 代码 / 模型 ID | `#ddd6fe` | 所有 monospace 内容（模型 ID、header 值） |
| Header 字段名 | `#94a3b8` | HTTP header key，次要 monospace 标签 |
| 描述 / 次要 | `#64748b` | 模型描述、说明文字 |
| 三级辅助 | `#475569` | 计数、模型数量、不重要的 meta 信息 |
| 极暗辅助 | `#334155` | context 标签文字、最低优先级元素 |

### 4.2 交互色 / 强调色

| 名称 | 色值 | 用途 |
|---|---|---|
| 主交互色 | `#6366f1` | 按钮、激活状态、保存/应用操作 |
| 主交互色亮版 | `#a5b4fc` | 选中状态文字、展开按钮激活色 |
| 警告 / 注意 | `#fbbf24` | ⚠ 提示图标、"智能" badge |
| 成功 | `#34d399` | 已保存、已配置状态 |
| 错误 | `#f87171` | 错误提示、未配置状态 |

### 4.3 服务商标识色

每个 AI 服务商有固定的标识色组（主色、背景、边框），定义在 `src/data/models.ts` 的 `PROVIDER_COLORS` 中。**不得在其他地方随意使用这些颜色**，仅限于分组按钮头部。

| 服务商 | 主色 |
|---|---|
| OpenAI | `#10b981`（绿） |
| Anthropic | `#f59e0b`（琥珀） |
| Gemini | `#6366f1`（靛蓝） |
| DeepSeek | `#38bdf8`（天蓝） |
| OpenRouter | `#ec4899`（粉） |

## 五、背景与边框

### 背景层次（由外向内叠加）

| 层次 | 背景值 | 说明 |
|---|---|---|
| 页面 | 全局深色（CSS 变量控制） | 最底层 |
| 卡片（Card） | `rgba(255,255,255,0.03)` 或类似极浅白 | 主容器 |
| 行项 / 子卡片 | `rgba(0,0,0,0.2)` | 列表行、设置项 |
| 选中 / 激活行 | `rgba(99,102,241,0.12)` | 当前选中的伪装模式等 |
| 代码块 / 深色区 | `rgba(0,0,0,0.25–0.35)` | 展开的 header 区域、代码展示 |

### 边框

| 场景 | 边框值 |
|---|---|
| 普通行项 | `1px solid rgba(255,255,255,0.05)` |
| 卡片 / 分组按钮 | `1px solid rgba(255,255,255,0.06–0.08)` |
| 激活 / 选中 | `1px solid rgba(99,102,241,0.4)` |
| 服务商分组 | 使用服务商 `PROVIDER_COLORS[provider].border` |
| OpenRouter 子分组 | `1px solid rgba(167,139,250,0.12)` |

## 六、圆角规范

| 元素 | 圆角 |
|---|---|
| 主卡片 | `12px` |
| 分组按钮 / 子卡片 | `8px` |
| 行项 | `6–7px` |
| 标签 / badge | `4–8px` |
| 按钮（小） | `5–6px` |
| 全圆（状态点） | `50%` |

---

## 七、间距规范

- 舒适密度（默认）：
  - 卡片内 padding：`16–24px`
  - 行项 padding：`7–10px 12–14px`
  - 行间 gap：`4–8px`（列表）
  - 卡片间 marginBottom：`16px`
  - 展开区域 padding：`8–10px 12–14px`
- 紧凑密度（`data-density="compact"`）：
  - 页面上下留白减少约 `25–30%`
  - 卡片内 padding 从 `24px` 降至 `18px`，从 `16px` 降至 `12px`
  - 行项垂直 padding 降至 `5–9px`
  - 常用 gap 从 `12/10/8px` 分别降至 `9/7/6px`
  - 文本行高降至 `1.35`，代码块行高降至 `1.45`
- 密度切换只调整信息密度，不改变主色、字号规范和组件层级。

## 八、组件约定

### 厂商名称规范

五大厂商名称固定如下，不得使用其他变体：

| 厂商标识符 | 显示名称 |
|---|---|
| `openai` | **OpenAI** |
| `anthropic` | **Anthropic** |
| `gemini` | **Google**（不得写为"Google Gemini"或"Gemini"） |
| `deepseek` | **DeepSeek** |
| `openrouter` | **OpenRouter** |

---

### 模型列表：基础模型与变体

模型分为**基础模型**（`isBase: true`）和**变体**两类，必须在 UI 中明确区分：

- **基础模型区分**：基础模型行**不显示任何文字标签**，仅以深色背景（`rgba(0,0,0,0.25)`）和较强边框（`1px solid rgba(255,255,255,0.07)`）与变体行做视觉区分
- **变体行**：背景略浅（`rgba(0,0,0,0.15)`），边框略淡（`rgba(255,255,255,0.04)`），以示区别
- **分组头计数**：显示「N 个基础模型 · M 个变体」；若无变体则省略变体部分；计数字号为 `12px`
- **变体范围**：变体包括思考（thinking）、思考可见（thinking-visible）、推理别名等所有非基础衍生版本，不仅限于思考变体
- **数据源**：`ModelEntry.isBase` 字段决定分类，必须在 `models.ts` 中为所有基础模型显式设置 `isBase: true`

---

### 折叠面板（collapsible）
- 默认只展示最重要的 2 个选项（如伪装模式仅展示「不伪装」和「自动」）
- 展开按钮放在列表底部，使用 `▾ / ▴` 前缀
- 展开内容通过 `border-left` 侧线与父级对齐（OpenRouter 子厂商样式）

### Headers 展开区
- 字段名（key）：`#94a3b8`，`minWidth` 固定（220px 或 200px）左对齐
- 字段值（value）：`#ddd6fe`，`wordBreak: break-all`
- 字号统一 `14px`，`Menlo, monospace`

### 状态 badge
- 已开启：`rgba(16,185,129,0.12)` 背景，`#34d399` 文字
- 已关闭 / 未配置：`rgba(100,116,139,0.15)` 背景，`#64748b` 文字
- 错误 / 未配置（醒目）：`rgba(239,68,68,0.1)` 背景，`#f87171` 文字

### 密度切换
- 入口放在 Header 右侧，与在线状态指示并列。
- 当前模式直接以文字显示：`舒适` / `紧凑`。
- 选择结果持久化到 `localStorage.portal_density_mode`，刷新页面后保持用户选择。
- 默认模式必须为 `舒适`，避免首次打开时信息过密。

---

## 九、禁止事项

- 不得使用 `12px` 以下字号用于正文内容
- 不得随意引入新颜色，所有新颜色必须来自本文档的调色板
- 不得在服务商分组以外使用服务商标识色（绿/琥珀/靛/天蓝/粉）
- 不得将主要内容行降到 `12px`；`12px` 仅用于次级信息、提示、标签和辅助元数据
- 不得让紧凑密度改变主字体大小；紧凑模式只调整 padding、gap、margin 和 line-height
- 不得破坏背景层次的叠加关系（越深层越暗）
- 不得将 Google 厂商显示为"Gemini"或"Google Gemini"，统一使用"Google"
- 不得在模型列表中仅统计思考变体；变体计数必须涵盖所有非基础衍生版本
